# Backend Completion Criteria

The backend is operational when:
1. Real public/port source writes latest snapshot.
2. Supabase stores normalized vessel master.
3. At least one snapshot history table is accumulating.
4. Candidate score can be recomputed from live data.
5. Sample records are blocked from commercial outreach.
6. Update pipeline completes within runtime budget.
7. Fallback snapshot is preserved on source failure.
