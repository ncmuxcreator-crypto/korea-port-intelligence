# Data Contract

This contract defines the minimum fields every dashboard API response must expose so the UI can distinguish real operating data from fallback diagnostics.

## Response envelope

Every API response should include these top-level fields when possible:

- `run_id`
- `active_run_id`
- `generated_at`
- `data_source_used`
- `fallback_used`
- `fallback_reason`
- `data_freshness`
- `record_count`

Static JSON fallback files should use `run_id` when they represent a concrete run. Worker/Supabase responses may use `active_run_id` and `summary_run_id`.

## Canonical ownership

- `vessel_master` owns vessel identity.
- `port_call_master` owns unique port visits.
- `vessel_snapshots` owns run-level state.
- `active_dataset_pointer` owns the live dashboard source.
- Daily snapshot tables own historical intelligence.

## Vessel row contract

Every vessel, candidate, opportunity, or port-call row should include these fields:

- `run_id`
- `generated_at` or `collected_at`
- `data_source_used` or source metadata in the parent response
- `port_call_id`
- `master_vessel_id`
- `vessel_name`
- `port_code`
- `port_name`
- `candidate_band`
- `commercial_value_score`
- `data_confidence_score`

Rows may contain null values when enrichment is incomplete, but the field name should remain present where the row is generated by the backend.

## Production rules

- `no_live_data` is diagnostics only.
- `no_live_data` must not promote `active_run_id`.
- `no_live_data` must not replace the latest successful dashboard summary.
- Empty static JSON files are acceptable only in local/no-secret diagnostics mode.
- Production mode requires `record_count > 0`, `all_vessels_count > 0`, and at least one successful Port Operation source.

## Dataset relationship

- `all_vessels`: all valid normalized vessel/port-call rows.
- `watchlist`: lower-priority commercial rows.
- `sales_targets`: qualified commercial targets.
- `immediate_targets`: highest-priority current or near-term rows.

`all_vessels` must never depend on candidate qualification.
